# Audit Report

## Title
Consensus DoS via Invalid Miner Order Values in Round Validation

## Summary
The AEDPoS consensus validation logic fails to verify that miner `Order` values in next round input are sequential integers starting from 1. A malicious miner can submit a `NextRoundInput` with all miners having `Order = 0` that passes validation but causes `GetMiningInterval()` to crash with `IndexOutOfRangeException`, resulting in complete blockchain halt.

## Finding Description

The vulnerability exists due to a critical mismatch between validation logic and runtime execution in the consensus contract.

**Validation Gap**: The `CheckRoundTimeSlots()` method orders miners by their `Order` field and validates time intervals between consecutive miners, but never validates that `Order` values are properly sequential (1, 2, 3, ...). [1](#0-0) 

If all miners have `Order = 0`, the validation passes as long as `ExpectedMiningTime` values have proper intervals.

**Runtime Crash**: The `GetMiningInterval()` method specifically filters miners where `Order == 1 || Order == 2` and accesses `firstTwoMiners[1]`. [2](#0-1) 

When no miners have Order values of 1 or 2, the filtered list is empty, causing an `IndexOutOfRangeException` when accessing index [1].

**Attack Path**:

1. A malicious miner calls `NextRound()` [3](#0-2)  with crafted `NextRoundInput` where all miners have `Order = 0` but properly spaced `ExpectedMiningTime` values.

2. Input is converted to Round via `ToRound()` which directly copies `RealTimeMinersInformation` [4](#0-3)  without validating Order values.

3. Validation runs through `ValidateBeforeExecution()` [5](#0-4)  which includes:
   - `TimeSlotValidationProvider` that calls `CheckRoundTimeSlots()` for new rounds [6](#0-5) 
   - `NextRoundMiningOrderValidationProvider` that only validates `FinalOrderOfNextRound`, not `Order` [7](#0-6) 
   - `RoundTerminateValidationProvider` that validates round number but not Order [8](#0-7) 

4. The malicious round passes validation and is stored [9](#0-8) 

5. When any node attempts to generate consensus commands via `GetConsensusCommand()` [10](#0-9) , the command strategies access `MiningInterval` property [11](#0-10)  which calls `GetMiningInterval()`, causing the crash.

6. Similarly, generating next round information crashes [12](#0-11)  because it also calls `GetMiningInterval()` at line 20.

## Impact Explanation

**Severity: CRITICAL - Complete Blockchain Halt**

Once the malicious round becomes the current consensus state, all nodes experience crashes when attempting core consensus operations:

- **Consensus Command Generation**: All nodes crash when calling `GetConsensusCommand()` to determine their next mining action, preventing any new block production.

- **Round Progression**: Any attempt to generate next round information crashes, preventing round transitions.

- **No Recovery Path**: The malicious round state persists in contract storage. Without a hard fork to reset consensus state, the chain cannot recover.

**Affected Parties**: 
- All validators cannot mine blocks
- All users cannot transact
- Network is permanently halted until manual intervention via hard fork

This breaks the fundamental availability guarantee of the blockchain, making it a critical consensus integrity violation.

## Likelihood Explanation

**Likelihood: HIGH**

**Attacker Requirements**:
- Must be a current or previous round miner (checked via `PreCheck()` [13](#0-12) )
- This is achievable by any participant in the validator set

**Attack Complexity: LOW**
- Craft `NextRoundInput` with all miners having `Order = 0`
- Set proper `ExpectedMiningTime` intervals (e.g., T, T+4000ms, T+8000ms, ...)
- Set proper `FinalOrderOfNextRound` values to match mined miners count
- Submit via `NextRound()` transaction when it's the attacker's turn to transition rounds

**Economic Cost**: Minimal - only one transaction fee. The attacker could even be the next scheduled extra block producer who earns rewards before executing the attack.

**Detection Difficulty**: The malicious transaction appears valid to all validation checks and is accepted by all nodes simultaneously, making pre-execution detection impossible.

**Motivation**: Any miner with grievance, desire to disrupt the network, or malicious intent can execute this attack.

## Recommendation

Add explicit validation in `CheckRoundTimeSlots()` or a dedicated validation provider to ensure `Order` values are properly sequential:

```csharp
public ValidationResult CheckRoundTimeSlots()
{
    var miners = RealTimeMinersInformation.Values.OrderBy(m => m.Order).ToList();
    if (miners.Count == 1)
        return new ValidationResult { Success = true };
    
    // NEW: Validate Order values are sequential starting from 1
    var expectedOrders = Enumerable.Range(1, miners.Count).ToList();
    var actualOrders = miners.Select(m => m.Order).OrderBy(o => o).ToList();
    if (!expectedOrders.SequenceEqual(actualOrders))
        return new ValidationResult { Message = "Invalid miner Order values - must be sequential 1..N" };
    
    if (miners.Any(m => m.ExpectedMiningTime == null))
        return new ValidationResult { Message = $"Incorrect expected mining time.\n{this}" };
    
    // ... rest of validation
}
```

Alternatively, add a dedicated validation provider for NextRound behavior that verifies Order field integrity.

## Notes

The vulnerability exploits the separation between validation-time checks (which validate `FinalOrderOfNextRound` and time intervals) and runtime operations (which depend on properly sequential `Order` values). The `Order` field is used for critical operations like calculating mining intervals and determining miner positions, yet is never validated to be within the required range [1..N] where N is the miner count.

    public ValidationResult CheckRoundTimeSlots()
    {
        var miners = RealTimeMinersInformation.Values.OrderBy(m => m.Order).ToList();
        if (miners.Count == 1)
            // No need to check single node.
            return new ValidationResult { Success = true };

        if (miners.Any(m => m.ExpectedMiningTime == null))
            return new ValidationResult { Message = $"Incorrect expected mining time.\n{this}" };

        var baseMiningInterval =
            (miners[1].ExpectedMiningTime - miners[0].ExpectedMiningTime).Milliseconds();

        if (baseMiningInterval <= 0)
            return new ValidationResult { Message = $"Mining interval must greater than 0.\n{this}" };

        for (var i = 1; i < miners.Count - 1; i++)
        {
            var miningInterval =
                (miners[i + 1].ExpectedMiningTime - miners[i].ExpectedMiningTime).Milliseconds();
            if (Math.Abs(miningInterval - baseMiningInterval) > baseMiningInterval)
                return new ValidationResult { Message = "Time slots are so different." };
        }

        return new ValidationResult { Success = true };
    }

    private ValidationResult ValidateBeforeExecution(AElfConsensusHeaderInformation extraData)
    {
        // According to current round information:
        if (!TryToGetCurrentRoundInformation(out var baseRound))
            return new ValidationResult { Success = false, Message = "Failed to get current round information." };

        // Skip the certain initial miner during first several rounds. (When other nodes haven't produce blocks yet.)
        if (baseRound.RealTimeMinersInformation.Count != 1 &&
            Context.CurrentHeight < AEDPoSContractConstants.MaximumTinyBlocksCount.Mul(3))
        {
            string producedMiner = null;
            var result = true;
            for (var i = baseRound.RoundNumber; i > 0; i--)
            {
                var producedMiners = State.Rounds[i].RealTimeMinersInformation.Values
                    .Where(m => m.ActualMiningTimes.Any()).ToList();
                if (producedMiners.Count != 1)
                {
                    result = false;
                    break;
                }

                if (producedMiner == null)
                    producedMiner = producedMiners.Single().Pubkey;
                else if (producedMiner != producedMiners.Single().Pubkey) result = false;
            }

            if (result) return new ValidationResult { Success = true };
        }

        if (extraData.Behaviour == AElfConsensusBehaviour.UpdateValue)
            baseRound.RecoverFromUpdateValue(extraData.Round, extraData.SenderPubkey.ToHex());

        if (extraData.Behaviour == AElfConsensusBehaviour.TinyBlock)
            baseRound.RecoverFromTinyBlock(extraData.Round, extraData.SenderPubkey.ToHex());

        var validationContext = new ConsensusValidationContext
        {
            BaseRound = baseRound,
            CurrentTermNumber = State.CurrentTermNumber.Value,
            CurrentRoundNumber = State.CurrentRoundNumber.Value,
            PreviousRound = TryToGetPreviousRoundInformation(out var previousRound) ? previousRound : new Round(),
            LatestPubkeyToTinyBlocksCount = State.LatestPubkeyToTinyBlocksCount.Value,
            ExtraData = extraData
        };

        /* Ask several questions: */

        // Add basic providers at first.
        var validationProviders = new List<IHeaderInformationValidationProvider>
        {
            // Is sender in miner list (of base round)?
            new MiningPermissionValidationProvider(),

            // Is this block produced in proper time?
            new TimeSlotValidationProvider(),

            // Is sender produced too many blocks at one time?
            new ContinuousBlocksValidationProvider()
        };

        switch (extraData.Behaviour)
        {
            case AElfConsensusBehaviour.UpdateValue:
                validationProviders.Add(new UpdateValueValidationProvider());
                // Is confirmed lib height and lib round number went down? (Which should not happens.)
                validationProviders.Add(new LibInformationValidationProvider());
                break;
            case AElfConsensusBehaviour.NextRound:
                // Is sender's order of next round correct?
                validationProviders.Add(new NextRoundMiningOrderValidationProvider());
                validationProviders.Add(new RoundTerminateValidationProvider());
                break;
            case AElfConsensusBehaviour.NextTerm:
                validationProviders.Add(new RoundTerminateValidationProvider());
                break;
        }

        var service = new HeaderInformationValidationService(validationProviders);

        Context.LogDebug(() => $"Validating behaviour: {extraData.Behaviour.ToString()}");

        var validationResult = service.ValidateInformation(validationContext);

        if (validationResult.Success == false)
            Context.LogDebug(() => $"Consensus Validation before execution failed : {validationResult.Message}");

        return validationResult;
    }

    public ValidationResult ValidateHeaderInformation(ConsensusValidationContext validationContext)
    {
        // Miners that have determined the order of the next round should be equal to

        var validationResult = new ValidationResult();

    public override ConsensusCommand GetConsensusCommand(BytesValue input)
    {
        _processingBlockMinerPubkey = input.Value.ToHex();

        if (Context.CurrentHeight < 2) return ConsensusCommandProvider.InvalidConsensusCommand;

            return ConsensusCommandProvider.InvalidConsensusCommand;

    public void GenerateNextRoundInformation(Timestamp currentBlockTimestamp, Timestamp blockchainStartTimestamp,
        out Round nextRound, bool isMinerListChanged = false)
    {
        nextRound = new Round { IsMinerListJustChanged = isMinerListChanged };

        var minersMinedCurrentRound = GetMinedMiners();
        var minersNotMinedCurrentRound = GetNotMinedMiners();
        var minersCount = RealTimeMinersInformation.Count;

        var miningInterval = GetMiningInterval();
        nextRound.RoundNumber = RoundNumber + 1;
        nextRound.TermNumber = TermNumber;
        nextRound.BlockchainAge = RoundNumber == 1 ? 1 : (currentBlockTimestamp - blockchainStartTimestamp).Seconds;

        // Set next round miners' information of miners who successfully mined during this round.
        foreach (var minerInRound in minersMinedCurrentRound.OrderBy(m => m.FinalOrderOfNextRound))
        {
            var order = minerInRound.FinalOrderOfNextRound;
            nextRound.RealTimeMinersInformation[minerInRound.Pubkey] = new MinerInRound
            {
                Pubkey = minerInRound.Pubkey,
                Order = order,
                ExpectedMiningTime = currentBlockTimestamp.AddMilliseconds(miningInterval.Mul(order)),
                ProducedBlocks = minerInRound.ProducedBlocks,
                MissedTimeSlots = minerInRound.MissedTimeSlots
            };
        }

        // Set miners' information of miners missed their time slot in current round.
        var occupiedOrders = minersMinedCurrentRound.Select(m => m.FinalOrderOfNextRound).ToList();
        var ableOrders = Enumerable.Range(1, minersCount).Where(i => !occupiedOrders.Contains(i)).ToList();
        for (var i = 0; i < minersNotMinedCurrentRound.Count; i++)
        {
            var order = ableOrders[i];
            var minerInRound = minersNotMinedCurrentRound[i];
            nextRound.RealTimeMinersInformation[minerInRound.Pubkey] = new MinerInRound
            {
                Pubkey = minersNotMinedCurrentRound[i].Pubkey,
                Order = order,
                ExpectedMiningTime = currentBlockTimestamp
                    .AddMilliseconds(miningInterval.Mul(order)),
                ProducedBlocks = minerInRound.ProducedBlocks,
                // Update missed time slots count of one miner.
                MissedTimeSlots = minerInRound.MissedTimeSlots.Add(1)
            };
        }

        // Calculate extra block producer order and set the producer.
        var extraBlockProducerOrder = CalculateNextExtraBlockProducerOrder();
        var expectedExtraBlockProducer =
            nextRound.RealTimeMinersInformation.Values.FirstOrDefault(m => m.Order == extraBlockProducerOrder);
        if (expectedExtraBlockProducer == null)
            nextRound.RealTimeMinersInformation.Values.First().IsExtraBlockProducer = true;
        else
            expectedExtraBlockProducer.IsExtraBlockProducer = true;

        BreakContinuousMining(ref nextRound);

        nextRound.ConfirmedIrreversibleBlockHeight = ConfirmedIrreversibleBlockHeight;
        nextRound.ConfirmedIrreversibleBlockRoundNumber = ConfirmedIrreversibleBlockRoundNumber;
    }
